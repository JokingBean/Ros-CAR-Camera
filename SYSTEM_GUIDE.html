二、核心方案

定位流程

步骤位置耗时说明
1. 抓图Pi + PC 并行~40msPiCam/USB1 在 Pi 上、USB2 在 PC 上同时抓
2. Tag 检测各自本地~30mspupil-apriltags 检测 tag36h11 小车 Tag (0,1,2,3)
3. PnP 求解各自本地~5mssolvePnP 计算 Tag 相对相机的位姿
4. 世界变换各自本地<1ms相机外参 (R,t) 将 Tag 位姿变换到世界坐标系
5. JSON 传输Pi→PC TCP~5ms仅传结果 (~200B)，不传图片 (~300KB)
6. GSD 融合PC<1ms按 1/GSD 加权平均各相机结果
总计~80ms≈ 12 Hz

融合公式

GSD = 相机到目标距离 / 焦距 × 1000  (mm/px，越小越精确)

权重_i = (1 / GSD_i) / Σ(1 / GSD_j)

融合位置 = Σ 权重_i × 位置_i
融合朝向 = Σ 权重_i × 朝向_i (水平投影归一化)

边缘为什么误差大

  • 距离远：远处 GSD 增大，每个像素覆盖更多地面
  • 相机少：仅1台相机时无融合校准，单相机偏差直接体现
  • 视角偏：Tag 在画面边缘时，角点检测精度下降
  • PnP 退化：Tag 平面和相机光轴接近平行时，朝向求解不稳定

五、关键设计决策

  • 不传图片，只传结果：Pi 端检测→JSON 回传，延时从 2s 降到 80ms
  • PiCamera 先开，USB 后开：避免 libcamera 驱动冲突
  • MJPG 格式：USB 高分辨率必须用压缩格式，YUYV 只能 640x480
  • bev 去畸变：BEV 投影前先 undistort，消除边缘拉伸变形
  • 摄像头保持打开：TCP 持续服务避免反复开关相机（每次省 ~500ms）
  • pipewire 冲突：Pi 上 pipewire 占用 /dev/video2，需 fuser -k 释放
