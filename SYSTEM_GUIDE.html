算法思路

第1步: 抓图+检测
每台相机各自拍一帧图像。
CLAHE (Contrast Limited Adaptive Histogram Equalization): 一种图像增强算法, 把画面中暗的地方提亮、亮的地方压暗, 让 Tag 的黑白格子更清晰, 更容易被检测到。
然后用 pupil-apriltags 库在图像中搜索小车上的 Tag (ID=0,1,2,3)。
输出: 每个 Tag 在图像中的 4 个角点坐标 (精度约 0.3 个像素)。
第2步: PnP求解Tag位姿
PnP (Perspective-n-Point): 已知物体在世界中的形状, 反推相机怎么拍的它。
这里已知: Tag 是边长 0.135m 的正方形, 以及它在图像中的 4 个角点位置。
OpenCV 的 cv2.solvePnP() 函数根据这 4 个点, 算出 Tag 相对相机的旋转和平移。
输出: 旋转矩阵 R (Tag 朝向) + 平移向量 t (Tag 位置, 单位: 米)。
第3步: 变换到世界坐标
上一步得到的是 "Tag 在相机坐标系中的位置"。我们需要的是 "Tag 在世界坐标系中的位置"。
外参 (R_w2c, t_w2c): 描述相机在世界中摆在哪儿、朝哪个方向, 通过标定提前算好。
用外参做一次坐标变换, 把 Tag 从相机坐标系转到世界坐标系。
同时算 GSD (Ground Sampling Distance): 在地面上, 一个像素代表多少毫米。
公式: GSD = 相机到 Tag 的距离 / 焦距 * 1000 (单位: mm/px)。
第4步: GSD加权融合
三台相机各自独立算出了一个 Tag 位置, 但精度不一样——哪个更可信?
GSD 越小 = 像素越密 = 看得越清楚 = 定位越准。
按 GSD 的倒数分配权重: 看得最清楚的相机, 话语权最大。
最终位置 = 各相机位置的加权平均值。
第5步: 立方体中心校正
Tag 贴在立方体的四个 侧面, 不是贴在中心。
立方体边长 25cm, 所以 Tag 中心到立方体几何中心差 12.5cm (半个边长)。
根据 Tag 贴在哪一面 (前/后/左/右), 沿相应方向偏移 12.5cm, 得到立方体的真实中心。

边缘误差大的根因

  • 距离远: GSD与距离成正比,4米外每像素覆盖6mm地面
  • 单相机: 无多相机交叉验证,单台偏差直接带入
  • 视角偏: Tag在画面边缘时透视变形大,角点检测精度下降
  • PnP退化: Tag平面接近平行于光轴时,深度方向求解不稳定

术语

缩写全称通俗解释
CLAHEContrast Limited Adaptive Histogram Equalization自动调亮度: 把暗处提亮、亮处压暗, 让 Tag 黑白格子更明显
PnPPerspective-n-Point已知物体的形状和它在照片中的样子, 反推它离相机多远、什么角度
GSDGround Sampling Distance地面上一个像素代表多少毫米。越小=看得越细=定位越准
RANSACRandom Sample Consensus排除异常点的算法: 试很多次, 挑大多数点都吻合的结果, 剔除少数"坏点"
外参Extrinsic Parameters描述相机在世界中的位置和朝向 (R=旋转, t=平移)
内参Intrinsic Parameters描述相机镜头的特性 (焦距 fx/fy, 光心 cx/cy, 畸变 k1/k2/p1/p2/k3)
BEVBird's Eye View鸟瞰俯视图: 把相机画面投影到地面上, 从正上方往下看
JSONJavaScript Object Notation一种轻量数据格式, 用于 Pi 和 PC 之间传结果 (~200字节)
