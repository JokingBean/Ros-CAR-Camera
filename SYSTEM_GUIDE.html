核心算法

1. 外参投影

P_cam = R · P_world + t
u = K[0,0] · x_cam / z_cam + K[0,2]
v = K[1,1] · y_cam / z_cam + K[1,2]

去畸变 → 投影到 BEV 平面 → 最近相机加权融合

2. Homography → 外参标定

H = λ · K · [r₁ r₂ t]
K⁻¹ · H → [r₁ r₂ t] → R = [r₁ r₂ r₁×r₂] → SVD 正交化 → solvePnP 微调

3. 立方体定位

solvePnP(Tag_四角点, K, dist) → R_tag, t_tag
Tag_世界坐标: tw = R_c²ʷ · t_tag + t_c²ʷ
立方体中心 XY = Tag XY（面偏移已省略）
融合: median({所有相机 × 所有 Tag 的 XY})

4. 融合权重

  • BEV 融合：1 / (1 + 距相机 BEV 中心距离 / 对角线/4)
  • 精度融合：1 / GSD（GSD 越小越精细）
  • 过滤：margin ≥ 20（AprilTag 解码可信度）

配置

config.yaml

cameras:
  - name: usb1      # Pi /dev/video0
    type: usb; host: pi; device: "0"
    resolution: [2560, 1440]; fps: 30
    camera_matrix: {fx: 1997.6, fy: 2004.4, cx: 1203.9, cy: 784.2}
    dist_coeffs: [0.08367, -0.15649, 0.00321, -0.00835, 0.11271]

  - name: usb2      # Pi /dev/video2  (相同内参)
  - name: usb3      # Pi /dev/video4  (相同内参)

extrinsics.yaml

usb1:
  R: [[r11, r12, r13], ...]
  t: [tx, ty, tz]

已知限制

1. 三台共享同一套内参
每台相机的实际焦距有差异。目前三台都使用旧 USB2 标定的 K (fx=1997, fy=2004)，导致定位存在系统偏差。需要每台单独做棋盘格标定才能稳定进入 2cm。
2. 面偏移已去除
Tag 在立方体面上距中心 12.5cm。解算的法线方向精度不够，12.5cm 偏移反而放大误差。当前直接用 Tag XY 作为立方体中心 XY，配合多面中位数抵消。
3. usb3 (idx=4) 亮度偏低
同一曝光参数下 mean≈110，低于 usb1/2 的 150-160。需额外提高 BRIGHTNESS 或 Gain。
4. margin < 20 观测自动丢弃
低可信度检测不参与融合，避免拉低精度。若所有观测都被过滤，会自动放行。
