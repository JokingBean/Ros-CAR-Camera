一、项目背景

1.1 V1 架构

树莓派 + USB 相机安装在小车顶部向下拍地面 AprilTag（9cm, 0.5m 网格, ~110 个）。solvePnP 解算小车位姿，与预设路径比对后输出纠偏指令，纯本地运行。

V1 优点： 纯本地、±1cm（Tag可见时）、有航向、有可视化。

1.2 V1 致命弱点

场景后果
画面模糊（震动/反光/运动模糊）Tag 检测失败 → 定位中断
Tag 出视野（转向/移动超出视野）丢失 → 纯递推 → 累积误差无界增长
遮挡（障碍物挡相机-Tag之间）全盲 → 无法纠偏
单相机单视角零冗余 → 一次失败即全局失败
本质：精度上限高（±1cm），但不保证下限——Tag 丢失就彻底失去控制。

三、BEV 俯视图融合（核心功能）

3.1 BEV 是什么

BEV（Bird's Eye View）俯视图将各相机斜视图像通过单应性变换投影到世界坐标系地面平面，生成从正上方往下看的鸟瞰图。三台相机的投影叠在一起，构成完整的场地覆盖。

用途：

  • 直观显示场地内所有地面 Tag 的位置和覆盖率
  • 每台相机的覆盖范围用不同颜色标注（USB1=红, USB2=绿, USB3=蓝）
  • Tag 圆点大小代表被几台相机同时看见（越大越亮 = 重叠越多 = 观测越可靠）
  • 实时作为 Web 追踪界面的背景底图，叠加小车位置和轨迹
三相机融合 BEV 俯视图

3.2 BEV 生成原理

3.2.1 外参标定 → 单应性矩阵

每台相机通过地面 AprilTag 标定得到外参 (R, t) —— 即相机在世界坐标系中的位姿。利用外参和内参 (K)，可以将地面平面上任意一点 (x, y, z=0) 投影到相机图像坐标 (u, v)：

P_c = R × [x, y, 0]ᵀ + t       # 世界 → 相机坐标系
(u, v) = K × P_c / P_c.z       # 相机 → 像素坐标

反过来，也建立了从图像像素到世界地面的映射（单应性）。

3.2.2 逐个像素逆映射

BEV 输出的每个像素对应世界地面上的一个点。对于 BEV 图上的像素 (u_bev, v_bev)，先算出它对应的世界坐标 (x, y, z=0)，再投影到各相机的图像坐标。如果落在相机视野内，就取该位置的像素颜色填到 BEV 图上：

for 每个 BEV 像素:
    x, y = BEV坐标 → 世界坐标(ppm像素/米换算)
    for 每台相机:
        (u_img, v_img) = 世界(x,y,0) → 相机投影
        if 落在图像内:
            BEV像素 = 相机图像(u_img, v_img) 的颜色

3.2.3 多相机重叠融合

同一个地面点可能被多台相机同时看到。融合策略采用最近相机优先（Nearest-Camera Priority）：

GSD (mm/px) = distance_to_point / focal_length × 1000

同一地面点，选择 GSD 最小的相机（即最近的相机）的像素，
因为 GSD 越小 → 分辨率越高 → 画面越清晰
USB1 原始图像 (2560×1440)
USB2 原始图像 (2560×1440)
USB3 原始图像 (2560×1440)
USB1 → BEV 投影（红橙色覆盖）
USB2 → BEV 投影（绿色覆盖）
USB3 → BEV 投影（蓝色覆盖）

四、GSD（地面采样距离）详解

4.1 GSD 定义

GSD = Ground Sampling Distance，即地面采样距离。它表示图像中单个像素在地面上对应的物理尺寸（mm/px）。GSD 越小，该位置的图像分辨率越高、越清晰。

4.2 GSD 计算公式

GSD (mm/px) = distance_to_point / focal_length × 1000

其中:
  distance_to_point = || R × P_world + t ||     (相机到目标点的 3D 距离)
  focal_length = (fx + fy) / 2                    (相机焦距, 像素单位)

推导:
  相似三角形: 地面尺寸 / 距离 = 像素尺寸 / 焦距
  → 地面尺寸 = 距离 × 像素尺寸 / 焦距
  → GSD = 距离 / 焦距  (像素尺寸归一化为1px)
  → ×1000 转换为 mm/px

4.3 GSD 的实际意义

相机距小车距离GSD 估算含义
usb1~3m~1.5 mm/px1 像素 = 1.5mm 地面
usb2~3.5m~1.8 mm/px1 像素 = 1.8mm 地面
usb3~2.5m~1.3 mm/px1 像素 = 1.3mm 地面（最好）

4.4 GSD 在系统中的应用

用途说明
BEV 融合选源同一地面点被多相机覆盖时，选 GSD 最小的相机像素
Tag 定位权重融合定位时，GSD 越小（越近、分辨率越高）的 Tag 权重大
冷启动代理权重无历史噪声数据时，用 GSD 映射为置信度 proxy 参与融合
Tag 质量评估GSD + margin + Tag 像素对角长度三指标联合评估检测质量
BEV 报告生成每个 Tag 的 GSD 按相机分别统计，生成 "Best Camera" 推荐

4.5 GSD 在代码中的位置

# pi_tracker.py: _process_detections() 中
P_cam = R @ tw + t                                        # 世界点 → 相机坐标系
dist_cam = norm(P_cam)                                     # 相机到 Tag 的距离
focal = (K[0][0] + K[1][1]) / 2                           # 平均焦距
gsd = dist_cam / focal * 1000.0                            # mm/px

# src/bev_engine.py: _gsd_at_point() 中
P_c = R @ P_w + t                                          # 世界点 → 相机坐标
dist = norm(P_c)                                           # 距离
focal = (K[0][0] + K[1][1]) / 2                            # 焦距
gsd = dist / focal * 1000.0                                # mm/px

# BEV 融合用 GSD 选最近相机:
gsd_by_cam = {cam: _gsd_at_point(tag_x, tag_y, 0, K, R, t)}
best_camera = min(gsd_by_cam, key=gsd_by_cam.get)         # GSD最小=最近相机

六、完整检测链路

阶段1: V4L2 参数预置

OpenCV 的 CAP_PROP_AUTO_EXPOSURE=1 在 Linux V4L2 上映射为手动模式（实为 Bug）。放弃 cap.set()，改用 v4l2-ctl 直写硬件：

v4l2-ctl -d /dev/video0 --set-ctrl=auto_exposure=3,white_balance_automatic=1,contrast=55,sharpness=55,gain=50

三相机用完全相同参数，打开后再设一次防止 OpenCV 重置。

阶段2: ROI 裁剪预测

线性外推预测下帧位置，仅裁剪 300×300 区域检测。检测区域从 3.7M 像素降至 90K（缩小 40 倍）。丢失时扩 200px，4 帧不回退全图。

阶段3: CLAHE + Tag 检测

clipLimit=2.5, tile=(8,8) 自适应直方图均衡增强对比度 → pupil-apriltags 检测 tag36h11。

阶段4: solvePnP → 世界坐标

solvePnP(obj_pts=13.4cm×13.4cm, corners, K, dist) → rvec, tvec (Tag→相机)
R_c2w = R.T;  t_c2w = -R_c2w @ t
tw = R_c2w @ t_tag2cam + t_c2w   # Tag中心在世界坐标

阶段5: 面修正航向

TAG_YAW_OFFSET = {0:-90°, 1:180°, 2:+90°, 3:0°}
车头航向 = median{各Tag.yaw + OFFSET[tag_id]} → EMA(α=0.3)

阶段6: 三层软加权融合

W = W_noise × W_dist × W_consist

W_noise:   100帧 σ → exp(-σ/3cm). 冷启动用GSD代理 (close→high weight)
W_dist:    高斯 σ=3cm, >6cm截断至1%
W_consist: Tag≥2→1.0, Tag=1→0.6

fused_xy = Σ(Wi × cam_xy_i) / ΣWi   (纯加权，不二值剔除)

阶段7: 渐进锁定平滑

3帧中位数前滤 → 偏差≤1cm锁死 / >1cm渐进修正(max 3cm/帧)

七、独立误差库

7.1 设计动机

三相机外参 5-15cm 系统偏差，各相机自身稳定但不一致。传统滤波无法区分"随机噪声"和"固定偏差"。

7.2 实现

结构: {camera_TtagID: {bx, by, n}}
更新: bx += 0.05 × bonus × (obs_x - ref_x), bonus∝观测数
应用: 输出坐标 - 该(cam,tag)的(bx, by)
锁死: 全局误差<1cm×30帧 → 冻结

十一、总结

V1：车载相机+地面Tag

  • ±1cm（看得见时）
  • 看不见→累积漂移
  • 1视角→零冗余
  • 遮挡→全盲

V2：固定相机+车载Tag

  • ±1cm 始终在线
  • 不丢：3视角×4面Tag
  • 多目冗余→抗遮挡
  • 50倍帧率(0.14→7.5Hz)
  • BEV融合+GSD精评

V2 不是更精准 —— 是更可靠 · 从「偶尔准」到「一直在」
