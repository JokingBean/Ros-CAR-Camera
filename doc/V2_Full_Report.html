ROS-Camera 多相机实时定位系统 — V2

从「间歇精准」到「始终在线」——完整技术报告

一、项目背景

1.1 V1 架构（车载相机 + 地面 AprilTag）

V1 版本中，树莓派 + USB 相机安装在小车顶部向下拍摄地面。地面铺设了约 110 个 AprilTag（9cm 边长，0.5m 间距网格）。系统通过 solvePnP 解算相机相对于 Tag 的位姿，得到小车在世界坐标系中的位置，与预设路径比对后输出纠偏指令。全程在树莓派本地运行，无需外部通信。

V1 优点： 纯本地运行、无网络依赖；Tag 清晰可见时定位精度达到 ±1cm；具备航向角输出和可视化界面。

1.2 V1 致命弱点

场景后果
相机画面模糊（震动/反光/运动模糊）Tag 检测失败 → 定位中断
Tag 出视野（小车转向/移动超出视野）完全丢失定位 → 纯靠递推 → 累积误差无界增长
遮挡（障碍物挡在相机和 Tag 之间）全盲 → 无法纠偏
单相机单视角零冗余 → 一次失败即全局失败
本质问题：V1 的精度上限很高（±1cm），但不保证下限——Tag 可见时定位优秀，一旦丢失就彻底失去控制，无法恢复。

五、核心技术原理

5.1 完整检测链路

从相机原始帧到最终定位结果，V2 的处理链路共 7 个阶段：

阶段 1：V4L2 硬件参数预置

OpenCV 的 CAP_PROP_AUTO_EXPOSURE=1 在 Linux V4L2 驱动上映射错误（实际设为手动模式而非自动）。因此放弃 OpenCV 的 cap.set()，改用 v4l2-ctl 直接写硬件寄存器。

v4l2-ctl -d /dev/video0 --set-ctrl=
  auto_exposure=1,exposure_time_absolute=60,
  white_balance_automatic=1,contrast=42,sharpness=48,gain=30

三台相机使用完全相同的硬件参数，消除帧间和相机间的亮度/色彩波动。AprilTag 检测对图像对比度和锐度高度敏感，硬件层面的统一是精度保证的第一道屏障。

阶段 2：ROI 裁剪预测

2560×1440 全图 AprilTag 检测耗时过高（100ms+），V2 用线性外推预测下一帧的车位置位置，只裁剪周边 300×300 像素区域进行检测。

vx = 当前位置 - 上一帧位置
vy = 当前位置 - 上一帧位置
pred_x = 当前位置 + vx
pred_y = 当前位置 + vy
ROI = pred_x,y 为中心 300×300 裁剪

检测区域：3,686,400 像素（全图）→ 90,000 像素（ROI），缩小 40 倍。丢失时每次扩大 200 像素，连续丢失 4 帧后回退全图扫描。

阶段 3：CLAHE 增强 + Tag 检测

裁剪区域转灰度后，经 CLAHE（Contrast Limited Adaptive Histogram Equalization，clipLimit=2.5, tile=(8,8)）增强对比度，再用 pupil-apriltags 库检测 tag36h11 系列的 AprilTag。

阶段 4：solvePnP 解算世界坐标

对每个检测到的 Tag，用其 4 个角点像素坐标 + 已知 Tag 物理尺寸（13.4cm） + 相机内参 K/dist，通过 OpenCV 的 solvePnP 解算 Tag 在相机坐标系下的位姿。再通过相机外参 (R, t) 转换到世界坐标系：

R_c2w = Rᵀ            # 相机→世界的旋转
t_c2w = -R_c2w @ t    # 相机→世界的平移
tw = R_c2w @ t_tag2cam + t_c2w  # Tag 中心在世界坐标

阶段 5：立方体面修正航向

solvePnP 输出每个 Tag 的 yaw 是该 Tag 面法线的方向，不是车头方向。4 个 Tag 在立方体的 4 个垂直面上，面法线各差 90°。将每个 Tag 的 yaw 加上面偏移后取中位数，得到真正的车头航向：

TAG_YAW_OFFSET = {T0(左面): -90°, T1(后面): 180°, T2(右面): +90°, T3(前面): 0°}
车头航向 = median{ 各Tag.yaw + TAG_YAW_OFFSET[tag_id] }
→ EMA(α=0.3) 平滑输出

阶段 6：三层软加权融合

V1 的二值"剔除/保留"机制导致相机进入/退出融合集合时位置跳变。V2 改为三层乘法权重，每路相机永远参与融合，不会突然消失：

W_total = W_noise × W_dist × W_consist

W_noise:  100帧滑动标准差 → exp(-σ / 3cm) 指数降权
          长期波动大的相机（如 usb2）自动降低话语权

W_dist:   高斯核 σ=3cm，偏离全局中位数 >6cm 截断至 1%
          单帧空间一致性：离群点自动衰减

W_consist: Tag ≥2 → 1.0, Tag =1 → 0.6
          单帧观测冗余：Tag 越多越可信

fused_xy = Σ(Wi × cam_xy_i) / ΣWi

冷启动期（前 200 帧）W_noise 用 GSD（地面采样距离）作代理：GSD 越小（相机越近）权重越高，避免无历史数据时权重均等带来的中点拉扯。

阶段 7：渐进锁定平滑

融合位置后接 3 帧中位数预处理（抹平单帧尖刺），再送入渐进锁定器：

偏差 = distance(候选点, 基准圆心)
if 偏差 ≤ 1cm:   输出 = 基准，不更新（锁死）
if 偏差 > 1cm:   输出 = 基准 + direction × min(3cm, 偏差×0.5)

效果：静止时零漂移，移动时无锯齿。

六、独立误差库（per-camera-per-tag 积分补偿）

6.1 设计动机

三台相机的外参标定存在 5-15cm 的系统偏差。同一位置下 usb1 报 (2.85, 2.07)、usb2 报 (2.90, 2.01)、usb3 报 (2.70, 1.93)——各相机自身稳定，但彼此不一致。传统滤波无法区分"随机噪声"和"固定偏差"。

6.2 实现原理

数据结构: { camera_TtagID: { bx, by, n } }

每帧更新:  bx += 0.05 × obs×bon × (观测_x - 平滑输出_x)
          收敛速度随观测数自动放大 (1× ~ 3×)
          限幅 ±10cm, 冷启动 5 帧后才启用

应用:     TAG_CENTER = 原始solvePnP输出 - 该(相机,Tag)的(bx, by)

稳态锁死: 全局平均误差 < 1cm 连续 30 帧 → 锁死所有偏差不再更新

6.3 关键约束

  • 不同相机/Tag 的偏差独立存储，互不干扰
  • 积分速度随观测数自适应调整，冗余充足时快速收敛
  • 锁死后彻底冻结，避免长期漂移
  • 与渐进锁定平滑配合：平滑提供基准（ref_x, ref_y），误差库不断补偿观测偏差

八、外参标定流程

8.1 标定原理

三台相机依次抓取 2560×1440 帧，检测地面 AprilTag（排除立方体 Tag ID 0-3），对每台相机用 solvePnPRansac 解算外参 (R, t)。标定在树莓派端本地执行，PC 端只下载结果文件。

十一、总结

V1：车载相机 + 地面 Tag

  • 看得见时：±1cm，精度很高
  • 看不见时：累积误差无界
  • 1 视角 → 零冗余
  • 遮挡 → 全盲

V2：固定相机 + 车载 Tag

  • 始终看得见：±1cm 始终在线
  • 不会丢：3 视角 × 4 面 Tag
  • 多目冗余 → 抗遮挡
  • 50 倍帧率（0.14→7.5 Hz）

V2 不是比 V1 更精准 —— 是比 V1 更可靠
