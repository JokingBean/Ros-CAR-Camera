瓶颈分析

SSH图片模式：瓶颈在 网络传输 + 相机开关
  PiCamera: Picamera2.start() 300ms + 下载300KB 1200ms = 1500ms
  USB1:     V4L2 open 200ms + 下载280KB 800ms = 1000ms
  总耗时: max(PiCam, USB1) + USB2 ≈ 1500 + 155 = ~7000ms (受串行影响)

TCP JSON模式：瓶颈在 相机开关
  Pi端: Picamera2保持open, 每次capture + detect ≈ 45ms
  总耗时: max(Pi端, USB2) ≈ 155ms
  帧率: 1000/155 ≈ 6.4 Hz (理论), 实测8-12 Hz

根本瓶颈: PiCamera.start() 每次300ms, 如果能保持open则降到~5ms

优化建议

  1. PiCamera保持open: 1332x990下start()只要150ms, 2028x1520需要300ms. 如果TCP服务器保持open可降到5ms
  2. 降分辨率: 1332x990足够检测Tag, 比2028x1520快2倍且传输体积小一半
  3. 并行传输: SSH串行下载PiCam+USB1共2200ms, 如果并行可降到max(1200,800)=1200ms
  4. JPEG压缩质量: cv2.imwrite默认95%质量, 降到80%可再减30%体积
